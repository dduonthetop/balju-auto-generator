# 연테이블 공동구매 기준파일 매핑 정의 (반복 운영용)

## 1. 작업 범위
- 타깃 파일: `연테이블 공동구매 <날짜>.xlsx` (예: `연테이블 공동구매 3월5일.xlsx`)
- 타깃 시트:
  - `발주서및송장번호` (상세 주문 라인)
  - `제품명 및 금액자료` (품목별 집계)
- 소스 파일 패턴: `주식회사지엠_*.xlsx` (시트명 `Supply`)

## 1-1. 실행 방식(고정)
- 실행 파일: `apply_mapping.py`
- 권장 실행 명령:
  - `python .\apply_mapping.py --target "연테이블 공동구매 <날짜>.xlsx" --source-date <YYYYMMDD>`
- 예시:
  - `python .\apply_mapping.py --target "연테이블 공동구매 3월5일.xlsx" --source-date 20260305`

## 2. 병합 규칙
- 소스는 `Supply` 시트의 2행부터 `주문번호`가 비어있지 않은 행만 사용한다.
- 여러 파일은 단순 세로 병합(append)한다.
- 중복 제거 키는 `주문상품고유번호`를 사용한다.  
  (`주문번호`는 상품별로 중복될 수 있으므로 중복 제거 키로 사용하지 않음)
- 전화번호/우편번호는 숫자 변환하지 말고 문자열 그대로 유지한다.
- 타깃 `발주서및송장번호` 시트에는 `A2:O`부터 순서대로 입력한다.
- `--source-date`를 지정하면 파일명에 해당 문자열이 포함된 `주식회사지엠_*.xlsx`만 반영한다.

## 3. `발주서및송장번호` 컬럼 매핑

| 타깃 컬럼 | 타깃 헤더 | 소스 컬럼 (`Supply`) | 매핑 규칙 |
|---|---|---|---|
| A | 보내는분 | (고정값) | `스미후루코리아` |
| B | 받는사람 | 수령인명 | 그대로 입력 |
| C | 받으시는분 전화 | 수령인연락처 | 그대로 입력 |
| D | 받는분 총주소 | 주소 | 그대로 입력 |
| E | 수량 | 주문수량 | 그대로 입력 |
| F | 품목명 | 고객선택옵션 | `고객선택옵션`을 `품명 및 규격` 규칙으로 변환한 값 입력 (예: `스미후루 감숙왕 바삭 바나나칩 70g x 8입` -> `바나나칩-08ea`) |
| G | 우편번호 | 우편번호 | 문자열 그대로 입력 |
| H | 특이사항 | 배송시 요청사항 | 그대로 입력 (비어있으면 공란) |
| I | 금액 | (계산) | `제품명 및 금액자료` 시트에서 `품명 및 규격`으로 품목명 조회 후 `공급가 × 수량` |
| J | 송장번호 | (고정 공란) | 항상 공란 |
| K | 주문번호 | 주문번호 | 그대로 입력 |
| L | 주문상품고유번호 | 주문상품고유번호 | 그대로 입력 |
| M | 상품코드 | 상품코드 | 그대로 입력 |
| N | 택배사 | (고정값) | `롯데택배` |
| O | 배송번호 | (고정 공란) | 항상 공란 |

## 4. `제품명 및 금액자료` 금액 참조 매핑

### 4.1 품목명 조회 기준
- 기준: 타깃 `발주서및송장번호`의 `품목명(F열)` 값을 `제품명 및 금액자료` 시트 `품명 및 규격(A열)`에서 찾는다.
- 일치 방식:
  - 1순위: 완전일치
  - 2순위: 완전일치가 없으면 포함일치(품목명 텍스트가 `품명 및 규격`에 포함되는 행)

| 타깃 행 | 타깃 품명(A열) | 소스 매핑 기준 |
|---|---|---|
| 3 | 바나밥 시즈닝 바나나칩 김맛-09ea | `고객선택옵션`에 `김맛` 포함 |
| 4 | 바나밥 시즈닝 바나나칩 샤워크림앤어니언맛-09ea | `고객선택옵션`에 `샤워크림` 또는 `어니언` 포함 |
| 5 | 바나밥 시즈닝 바나나칩 솔티드카라멜맛-09ea | `고객선택옵션`에 `솔티드카라멜` 포함 |
| 6 | 바나밥 시즈닝 바나나칩 3종(어니언1+솔티드1+김1)-03ea | `상품코드 = SOEN24854874` 또는 옵션에 `3종 혼합` 포함 |
| 7 | 쿠키슈 4입-02ea | `상품코드 = SOEN85705636` 또는 옵션에 `쿠키슈` 포함 |
| 8 | 바나나칩-08ea | `상품코드 = SOEN92173485` 또는 옵션에 `바삭 바나나칩` 포함 |

추가 정규화 규칙:
- 옵션 문자열 비교 시 공백/`/`/괄호/하이픈/언더스코어/점(`.`)은 제거 후 비교한다.
- 예시: `3종혼합`, `솔티드 카라멜`, `감숙왕 바나나 쿠키슈`, `바나나칩70gx8입`도 기존 규칙과 동일하게 매핑한다.

### 4.2 입력 규칙
- H열(발주수량): 위 기준으로 `주문수량` 합계를 입력한다.
- I열(금액): `I행 = G행 * H행` 공식으로 계산한다.
- 상세 시트(`발주서및송장번호`) 금액(I열): `제품명 및 금액자료`에서 조회한 공급가 × 해당 행 수량(E열)
- TTL(9행) 권장:
  - `H9 = SUM(H3:H8)`
  - `I9 = SUM(I3:I8)`

## 4-1. 서식 규칙(확정)
- 1행(헤더)은 기존 서식 유지.
- 2행 이후 `A:O`는 모두 동일 본문 스타일로 적용.
- 2행 이후 `A:O` 셀 배경색은 전부 `없음`으로 강제 적용.
- 금액(I열) 표시형식은 원화 천단위 쉼표: `₩#,##0` (예: `14,500`).

## 4-2. 수량 분할 규칙(확정)
- `발주서및송장번호` 상세 라인은 `주문수량`만큼 행을 분할해서 작성한다.
- 분할된 각 행의 `수량(E열)`은 항상 `1`이다.
- 같은 주문자가 2개 구매하면 2행, 3개 구매하면 3행으로 나열한다.
